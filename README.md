# Pallets-Site

Marketing website work for **Pallets, Pallets, Pallets, LLC** (`palletspalletspallets.com`).

This repo holds an **example static marketing site** that demonstrates a properly-SEO'd replacement for the Client's current website-builder site. It's a working reference — not a contracted deliverable. Stack: Astro 5 + Tailwind 4, static build, Cloudflare-Pages-ready.

The active in-flight work for Anthony's existing site is the SEO quick-fix scope (titles, meta, H1, alt text, LocalBusiness JSON-LD, service-area paragraph, Search Console). That scope is bundled into the platform-pilot engagement as goodwill — see the related repo below.

## Quick start

```bash
npm install
npm run dev          # http://localhost:4321
npm run build        # static output to dist/
npm run preview      # serve dist/ for QA
```

## Pages

- `/` — Home (hero, services overview, service-area teaser, CTAs)
- `/services` — Pallet supply, pickup & recycling, custom, Grade A/B
- `/service-area` — 4-paragraph long-form service-area copy + city list (the audit's 300–500 word requirement, expanded)
- `/about` — Company narrative, how-we-work steps
- `/contact` — Phone, email, yard address, hours, what-to-tell-us checklist

## SEO controls baked in

- `<title>` and `<meta name="description">` per page via `src/components/SEOHead.astro` (warns at build time if a page exceeds 60 / 155 chars)
- One `<h1>` per page, semantic heading structure throughout
- `alt` on every meaningful image; decorative SVGs marked `aria-hidden`
- `LocalBusiness` JSON-LD on every page via `src/components/LocalBusinessSchema.astro` (name, address, phone, email, hours, areaServed GeoCircle, knowsAbout)
- Canonical URLs, Open Graph, Twitter Card on every page
- `sitemap-index.xml` + `sitemap-0.xml` generated by `@astrojs/sitemap` on build
- `robots.txt` pointing to the sitemap
- Skip-to-main-content link, focus-visible styles, semantic landmarks for accessibility

## Configuring for production

Site-wide content (name, phone, email, address, hours, service area, city list, nav links) is centralized in `src/data/site.ts`. Replace placeholder values before deploying:

- `phone` and `phoneDisplay` — currently a `555-0000` placeholder
- `email` — currently `info@palletspalletspallets.com` (generic, not Anthony's direct address)
- Hours and weekend hours
- Service-area radius and city list

The address in `site.ts` matches what's already on the existing site's public records; double-check before deploying.
